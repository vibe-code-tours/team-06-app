Customer order flow — scope confirmed

  • Customer builds cart (add / remove / adjust qty) → submits order → real-time status. This lives inside #03 Order Management.
  • Payment stays cashier-mediated (customer flags bill, cashier confirms payment) — no self-service online checkout in scope for now.

One dev has to own Supabase dashboard + Repository

repo-wide
This person owns the Supabase dashboard — holds the service role key, is the only one who runs migrations to production, reviews any PR touching supabase/migrations/. Everyone else changes schema/RLS only via migration file + PR, never the dashboard directly, or dashboard state and git drift apart.

Cross-cutting — not owned by a single epic

CC-1

Documentation

per-epic + root
Each dev documents their own epic (API contract, setup notes) as they build it. Root README.md / CONTRIBUTING.md owned by whoever scaffolds.
CC-2

Manual flow / integration testing

at dependency handoffs
The downstream epic owner manually verifies against the upstream epic before building on it (e.g. #03 checks #02's API; #06 checks #07's session functions).
CC-3

End-to-end scenario testing

full journey
Scan → cart → order → kitchen → payment, tested as one continuous flow once #02, #03, #06, #07 are far enough along. Best as a shared "integration day" rather than one person's task.
CC-4

Deployment / CI

repo-wide
One pipeline/config for the whole repo — U Lay provided in team repo.
CC-5

Role & Permission Management

every epic, own column
Spec'd as its own feature in feature-spec.md §11 (7 roles × full permission matrix, RLS policy summary, 3-layer enforcement: middleware → API → DB). Not a separate dev's job — each epic owner implements their own column of the matrix as they build (e.g. #04 Kitchen enforces "no payment access" themselves). Easy to silently skip if it isn't checked per-epic.

Code review

  • No single reviewer — everyone reviews everyone's PRs.
  • Keep an eye on packages/shared changes specifically, since all 9 epics import from it — flag those PRs for a second look before merging.
